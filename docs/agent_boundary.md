# Agent Boundary for Zemax Automation

## 1. AI Agent 的职责

AI Agent 负责：

1. 理解用户的自然语言仿真需求。
2. 提取模型文件、扫描参数、参数范围、评价指标和输出要求。
3. 将需求整理成结构化任务。
4. 检查参数是否缺失、单位是否明确、范围是否合理。
5. 在仿真结束后，根据 CSV 和图表总结结果趋势。
6. 生成报告草稿和 README 描述。

## 2. AI Agent 不负责的事情

AI Agent 不负责：

1. 不直接操作 Zemax 软件界面。
2. 不直接修改原始 Zemax 模型文件。
3. 不凭空生成仿真结果。
4. 不自行决定超大范围参数扫描。
5. 不跳过 ZOS-API / Python 脚本。
6. 不把没有数据支持的内容写成结论。

## 3. Python / ZOS-API 的职责

Python / ZOS-API 负责：

1. 打开指定 Zemax 模型。
2. 读取和修改 LDE 中的参数。
3. 运行 MTF、Spot、Ray Fan 等分析。
4. 导出 CSV、PNG、TXT 或新模型文件。
5. 记录日志。
6. 返回执行结果给 AI 总结。

## 4. 安全原则

1. 原始模型只读。
2. 所有输出保存到 results/ 目录。
3. 所有参数必须有单位。
4. 所有扫描范围必须设置上限。
5. 运行前先 dry-run，确认任务数量。
6. AI 的总结必须基于真实导出的 CSV 和图表。
